01 / 核心

先记住一句话
布局决定元素彼此的位置关系

核心

字号、颜色、边框，是在管“长什么样”；这一课先看页面怎么拆成几个区域，谁放在上面，谁在左边，谁占主体。

02 / 传统门户 / 官网

传统门户 / 官网骨架
先把入口、主视觉和内容区分开

重点

典型结构是 header + nav + banner + main + footer。适合首页、活动页、产品官网，重点是先把入口、主视觉和内容区分开。

Pattern 01

传统门户 / 官网布局

HTML 大概长这样 
<header>
  <nav>Logo / 导航 / 登录</nav>
</header>
<main>
  <section class="banner">主视觉</section>
  <section class="cards">
    <article>功能卡片 1</article>
    <article>功能卡片 2</article>
    <article>功能卡片 3</article>
  </section>
</main>
<footer>版权 / 联系方式</footer>
可以这样描述

做一个门户首页：顶部是 header，里面放 Logo 和导航；中间是大 banner 和几块功能卡片；下方是 footer。桌面端多列，移动端收成单列。

关键词：header、nav、banner、main、footer

03 / 后台管理系统

后台管理系统骨架
左侧导航，右侧主体

重点

结构上通常是 header + aside + main，其中 aside 贴最左并铺满高度，main 吃掉剩余空间。

Pattern 02

后台管理系统布局

HTML 大概长这样 
<header>系统头部</header>
<div class="layout-shell">
  <aside>左侧导航</aside>
  <main>
    <section>筛选栏</section>
    <section>表格 / 图表 / 表单</section>
  </main>
</div>
可以这样描述

做一个后台管理系统：顶部是 header；左侧是固定宽度的 aside 导航，贴最左并铺满整条高度；右侧 main 作为主体区域，展示表格、图表和表单；移动端把侧边栏收起。

关键词：header、aside、main、固定宽度、占据主体

04 / 博客 / 文章详情

博客 / 文章详情页骨架
正文在中间，侧栏在旁边

重点

常见结构是 header + article + aside + footer。主内容是文章，右侧是作者信息、目录和相关文章。

Pattern 03

博客 / 文章详情页布局

HTML 大概长这样 
<header>博客导航 / 搜索</header>
<main>
  <article>
    <h1>文章标题</h1>
    <p>正文内容...</p>
  </article>
  <aside>
    <section>作者信息</section>
    <section>目录</section>
    <section>相关文章</section>
  </aside>
</main>
<footer>版权 / 备案</footer>
可以这样描述

做一个博客详情页：顶部是 header；中间左边是文章正文 article，右边是 aside，放作者信息、目录和相关文章；底部是 footer；移动端把右侧栏折到下方。

关键词：article、aside、目录、footer、单列

05 / 常见问题

先把最容易混淆的一件事说清楚：
左右位置主要由 CSS 决定。

为什么写了 aside，有时在左边，有时在右边？

aside 只是语义标签，本身不决定左右。真正决定它在哪一侧的是父容器的 CSS：如果用了 flex，就看 flex-direction、order、以及主轴和交叉轴怎么定义；如果用了 grid，就看列怎么分配。HTML 负责表达“这里是 aside”，CSS 负责决定它是左边、右边，还是在下方。

那移动端为什么经常又会变成上下排列？

因为在窄屏下，CSS 往往会把原来的两列布局改成单列，最常见的做法是把 flex-direction 从横向改成纵向，或者直接重设 grid-template-columns。这样正文优先展示，aside 再往下堆叠。
