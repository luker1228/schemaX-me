# Luke IP

## 角色定义

Luke 是 Ian 中文正文配图的固定视觉 IP，参考图见 `assets/ip/luke-ip.png`。

默认每张图都要出现 Luke。Luke 不是吉祥物，不是贴纸，不是可爱装饰，而是正在认真参与系统运转的主角。

## 外形

- 男孩形象，大头小身体比例。
- 黑色毛寸短发，发梢有一点凌乱感。
- 大号黑框眼镜，眼睛圆，视线专注。
- 圆圆的偏粉鼻头，脸颊有一点红晕。
- 常穿宽松蓝色卫衣，胸口挂着黄色的'luke'标签。
- 常见姿态是盘腿坐着、趴着看、俯身操作、抱着电脑或本子思考。
- 气质是安静、认真、稍微发愁，但不苦大仇深。

## 性格

- 很认真，但做的事有点荒诞。
- 像一个长期写代码、写文章、搭工作流的人。
- 有冷幽默，但不卖萌。
- 可以轻微笨拙，但核心气质是专注和靠谱。
- 像在白纸草图里真的负责某个工作台。

## 常见职责

让 Luke 承担核心动作：

- 搬运素材。
- 拉线汇聚信息源。
- 卡在断点里排查问题。
- 在机器里操作“判断”杆。
- 对着电脑、终端、卡片墙、抽屉或怪机器做筛选。
- 拆解“素材鱼”或把一个素材压成几种输出。
- 盖章承接话术。
- 牵着承接路径。
- 举警告牌看坑。
- 从洞里伸手但接不住内容。
- 在旁边搬砖、搭桥、开门、分拣、记录。

## 禁止

- 不要把 Luke 画成过度可爱的吉祥物。
- 不要画成儿童绘本主角或 Q 版海报。
- 不要把五官做得太夸张，尤其不要闪亮大眼、夸张笑脸。
- 不要加复杂时装、潮牌感或精致商业插画质感。
- 不要让 Luke 只是站在角落里看。
- 不要让 Luke 抢走结构表达。

## 判断标准

如果去掉 Luke，图的核心隐喻还能完全成立，说明 Luke 太装饰了；要重写提示词，让 Luke 成为动作主体。
